# Security Policy for Courses Algérie

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public issue**. Instead, contact the project maintainer directly:

- **Email:** [dzmohamedali31@gmail.com](mailto:dzmohamedali31@gmail.com)  

Include as much detail as possible so the issue can be reproduced and fixed safely. 

---

## Supported Versions

This security policy applies to the latest stable version of Courses Algérie. Older versions may not receive security updates. 

---

## Responsible Disclosure

We follow responsible disclosure practices:

1. Please report any vulnerability privately first.  
2. Give us reasonable time to fix the issue before publicly disclosing it.  
3. We will acknowledge contributors who report valid vulnerabilities, unless they wish to remain anonymous.  

---

## What Not to Do

- Do **not** attempt to exploit vulnerabilities on live systems.  
- Do **not** publicly disclose vulnerabilities before a fix is available.  

---

## Security Best Practices for Contributors

If you contribute to the repository:

- Do **not** commit any secrets, passwords, or private keys.  
- Use `.env` files for credentials and add them to `.gitignore`.  
- Check dependencies for security vulnerabilities before adding new packages.

---

Thank you for helping us keep Courses Algérie safe!
